# <a name="engage-through-apps"></a>Привлечение клиентов с помощью приложений

::: zone-end

Инновации включают в себя модернизацию существующих приложений, размещенных в локальной среде, а также создание ориентированных на облако приложений с помощью контейнеров или бессерверных технологий. Для модернизации приложений Azure предоставляет службы PaaS, такие как Служба приложений Azure, позволяющие легко модернизировать существующие веб-приложения и приложения API, написанные на .NET, .NET Core, Java, Node.js, Ruby, Python или PHP, и развертывать их в Azure. Модель контейнеров с открытым стандартом позволяет создавать микрослужбы или контейнеризировать существующие приложения и развертывать их в Azure. Это с легкостью можно сделать с помощью управляемых служб, таких как Azure Kubernetes, "Экземпляры контейнеров Azure" и Веб-приложения для контейнеров. Бессерверные технологии, такие как Функции Azure и Azure Logic Apps, позволяют сосредоточиться на создании приложения с использованием модели потребления (оплата только за то, что используется) вместо развертывания инфраструктуры и управления ею.

## <a name="azure-app-service"></a>Служба приложений Azure

Служба приложений Azure предоставляет среду размещения для приложений, что устраняет нагрузку в контексте управления инфраструктурой и исправления ОС. Она обеспечивает автоматизацию масштабирования в соответствии с потребностями пользователей и с учетом ограничений для сохранения уровня затрат.

Служба приложений Azure полностью поддерживает такие языки, как ASP.NET, ASP.NET Core, Java, Ruby, Node.js, PHP и Python. Если вам нужно разместить другой стек среды выполнения, компонент "Веб-приложение для контейнеров" позволяет быстро и легко разместить контейнер Docker в среде Службы приложений Azure, тем самым размещая пользовательский стек кода вне серверной бизнес-среды.

## <a name="what-are-cloud-native-applications"></a>Что такое ориентированные на облако приложения?

Ориентированные на облако приложения разработаны с нуля и оптимизированы для работы в облаке. Они слабо связаны и основаны на архитектуре микрослужб, используют управляемые службы, доступны для отслеживания, а также обеспечивают надежность и быстрый выход на рынок за счет непрерывной поставки. Обычно они переносимы и могут работать в динамических средах, таких как общедоступные, частные и гибридные облака. Ориентированные на облако приложения обычно создаются с помощью одного или нескольких следующих подходов:

- Микрослужбы
- Бессерверные приложения
- Контейнер

## <a name="microservices"></a>Микрослужбы

Микрослужбы — это стиль архитектуры программного обеспечения, в котором приложения состоят из небольших независимых модулей, которые обмениваются данными друг с другом с помощью четко определенных контрактов API. Эти служебные модули представляют собой полностью несвязанные строительные блоки, которые достаточно малы для реализации единой функциональности. Микрослужбы помогают в следующих ситуациях:

- создание служб независимо друг от друга;
- масштабирование служб автономно;
- использование наиболее подходящих подходов к развертыванию и языку программирования;
- изолирование точек сбоя;
- оперативное повышение рентабельности.

### <a name="azure-kubernetes-service-aks"></a>Служба Azure Kubernetes (AKS)

Полностью управляемую службу Kubernetes можно использовать для обработки, обновления и масштабирования ресурсов кластера по требованию. Служба AKS упрощает развертывание контейнерных приложений и управление ими. Она предоставляет бессерверную платформу Kubernetes со встроенными возможностями непрерывной интеграции и непрерывной поставки,а также функции безопасности и управления корпоративного уровня. Обеспечьте совместную работу специалистов по разработке и эксплуатации на единой платформе, позволяющей оперативно и уверенно создавать, доставлять и масштабировать приложения.

### <a name="azure-functions"></a>Функции Azure

Решение "Функции Azure" предоставляет платформу для запуска небольших фрагментов кода (функций) в облаке. Функции помогут при выполнении рефакторинга кода в архитектуру микрослужб.

Среда выполнения Функций Azure поддерживает множество языков, включая C#, Java, JavaScript и Python. Дополнительные сведения см. в статье о [поддерживаемых языках в решении "Функции Azure"](https://docs.microsoft.com/azure/azure-functions/supported-languages).

Еще одним преимуществом функций является возможность их запуска различными действиями и событиями, такими как триггеры HTTP, триггерами-таймерами и триггерами из других служб Azure, таких как хранилище BLOB-объектов, Сетка событий и Служебная шина. См. дополнительные сведения о [триггерах и привязках в решении "Функции Azure"](https://docs.microsoft.com/azure/azure-functions/functions-triggers-bindings).

## <a name="serverless-solutions"></a>Бессерверные решения

- повышение скорости разработки;
- повышение производительности команды;
- улучшение организационного воздействия.

## <a name="azure-front-door"></a>Azure Front Door

Azure Front Door обеспечивает масштабируемую, безопасную точку входа для предоставления приложения по всему миру. Эта служба сочетает оптимизацию трафика для лучшей производительности и мгновенную глобальную отработку отказа. Azure Front Door будет полезнее диспетчера трафика, если вам необходима обработка подключений по протоколу TLS (разгрузка SSL) или обработка прикладного уровня HTTP- или HTTPS-запросов.

### <a name="action"></a>Действие

Чтобы настроить или отслеживать Front Doors, выполните следующие действия:

1. Щелкните **Front Door**.
2. Настройте новую службу Front Door, щелкнув ссылку **Добавить +** и следуя инструкциям на экране.
3. Управление существующими службами Front Door: Выберите нужную службу Front Door из списка.

## <a name="traffic-manager"></a>Диспетчер трафика

Диспетчер трафика обеспечивает балансировку нагрузки на основе DNS, которую можно маршрутизировать на основе различных правил. Это помогает обеспечить устойчивость в случае сбоя развернутых служб. Диспетчер трафика можно также использовать для маршрутизации на основе сбоев и маршрутизации на основе производительности, предоставляя лучшие возможности с учетом географического расположения.

### <a name="action"></a>Действие

Чтобы настроить или отслеживать профили диспетчера трафика, выполните следующие действия:

1. Щелкните **Профили диспетчера трафика**.
2. Настройте новый профиль, щелкнув ссылку **Добавить +** и следуя инструкциям на экране.
3. Управление существующими профилями: Выберите нужный профиль из списка.

## <a name="azure-content-delivery-network"></a>Сеть доставки содержимого Azure

Azure предлагает распределенную сеть доставки содержимого (CDN), которая позволяет своевременно предоставлять ресурсы, кэшируя их рядом с расположением пользователей. Такое кэширование помогает улучшить взаимодействие с клиентами и предотвращает проблемы при загрузке содержимого, вызванного сетевыми проблемами между конечной точкой CDN и центром обработки данных, в котором размещается ваше приложение. Эта сеть CDN также может использоваться приложениями, не размещенными в Azure.

### <a name="action"></a>Действие

Чтобы настроить или отслеживать профили CDN, выполните следующие действия:

1. Щелкните **Профили CDN**.
2. Настройте новый профиль, щелкнув ссылку **Добавить +** и следуя инструкциям на экране.
3. Управление существующими профилями: Выберите нужный профиль из списка.
