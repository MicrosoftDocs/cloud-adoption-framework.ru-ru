# <a name="empower-adoption"></a>Поддержка внедрения

::: zone-end

Организации понимают, что инновации очень важны для успеха бизнеса. Инновации не внедряются исключительно путем введения новых технологий. Вместо этого нужно уделять внимание поддержке тех людей, которые катализируют изменения и создают новые ценности для организации. **Разработчики** являются двигателями цифрового преобразования, и для расширения их возможностей нужно повысить скорость разработки. Чтобы высвободить творческую энергию команд разработчиков, необходимо повысить эффективность их труда, обеспечить глобальную и безопасную совместную работу, а также устранить барьеры для масштабирования инноваций.

## <a name="value-generation"></a>Создание ценностей

> [!div class="checklist"]
>
> - Независимо от отрасли, каждая организация пытается стимулировать постоянный рост ценностей.
> - Внимание к инновациям — это, по сути, процесс, позволяющий организациям находить новые способы создания ценности.
> - Возможно, самая большая ошибка организаций заключается в повторяющихся попытках создать ценности с оглядкой на технологии.
> - Иногда преобладающее мнение звучит так: "давайте просто используем больше технологий и подождем улучшений". Тем не менее, инновации в первую очередь зависят от людей.
> - Инновации — это сочетание людей и технологий.

Организации, которые успешно внедряют инновации, рассматривают концепцию, стратегию, культурологические характеристики, уникальный потенциал и возможности в качестве базовых элементов, а затем используют технологии с определенной целью. Каждая компания становится зависимой от программного обеспечения. По данным LinkedIn, уровень востребованности разработчиков программного обеспечения растет с более высокой скоростью за пределами отрасли высоких технологий, чем внутри.

Инновации достигаются, когда организации поддерживают сотрудников в процессе создания нужной ценности. Одной из групп людей, которые являются катализаторами инноваций, являются разработчики. Разработчики играют все более важную роль в процессе роста и создания ценности в каждой отрасли. Разработчики — это творцы нашей эры, пишущие код этого мира и движущие процесс инноваций. Инновационные организации формируют культуру, в которой разработчики могут добиться большего.

# <a name="developer-productivitytabdeveloperproductivity"></a>[Продуктивность разработки](#tab/DeveloperProductivity)

## <a name="developer-velocity"></a>Скорость разработки

Предоставление разработчикам возможностей для инноваций означает повышение скорости их работы, благодаря чему они могут делать больше разработок, внедрять больше инноваций и решать больше проблем. Скорость разработки — это основа технического развития каждой организации. Скорость разработки заключается не только в темпе, но и в свободе проявлять изобретательность &mdash; превращать идеи в программное обеспечение с максимальной скоростью и гибкостью, что способствует созданию инновационных решений. Дифференцированное решение Azure предназначено для стимулирования инноваций в вашей организации.

## <a name="build-productively"></a>Эффективная разработка

Существует две области возможностей, в которых Azure может способствовать эффективной разработке:
  
> [!div class="checklist"]
>
> - Поддержка повышения компетентности разработчиков в их области, помогая им расширять свои знания.
> - Стимулирование развития нужных навыков сотрудников, предоставив им нужные средства.

Один из лучших способов улучшить навыки разработчиков — предоставить им знакомые и удобные средства. Средства Azure соответствуют требованиям разработчиков и представляют им новые технологии в контексте кода, с которым они работают. Благодаря поддержке всех языков и платформ и возможности использовать ПО с открытым кодом средства Azure позволяют разработчикам создавать приложения удобным для них способом и развертывать их в нужных средах.

Набор Azure DevOps предоставляет лучшие в своем классе средства для каждого разработчика. Службы разработчики Azure применяют современные методики разработки и новые тенденции в наших средствах. Благодаря платформе Azure разработчики получают доступ к новейшим технологиям, а также к высоко эффективной цепочке инструментов, подходящей для любых методов работы.

> [!div class="checklist"]
>
> - Средства разработки на базе искусственного интеллекта
> - Интегрированные средства и облако
> - Удаленная разработка и парное программирование

# <a name="innovate-collaborativelytabcollaboration"></a>[Совместное внедрение инноваций](#tab/Collaboration)

## <a name="innovate-through-collaboration"></a>Внедрение инноваций с применением совместной работы

Разработка программного обеспечения — это командный спорт. Совместная работа с коллегами и обмен знаниями в команде очень важны, как и устранение барьеров между разработкой и операциями. Существует также множество знаний за пределами основной команды разработчиков, которые можно получить на таких платформах, как GitHub. GitHub, самое большое сообщество разработчиков, — это центр сообщества по работе с открытым кодом, которая всегда окажет поддержку всем разработчикам. Совместная работа лежит в основе GitHub &mdash; в течение года разработчики сделали более 1 млн вкладов в проекты с открытым кодом и частные репозитории, размещенные на сайте GitHub. Здесь разработчики могут совместно работать, хранить свой исходный код и управлять непрерывной интеграцией.

**Перейдите к [Microsoft Forms](https://docs.microsoft.com/MicrosoftTeams)** .

## <a name="open-source-innovation"></a>Инновации с открытым кодом

Для создания инноваций многие разработчики используют открытый код. Около 80–90 % разработки новых приложений начинается с открытого кода. Открытый код — это не только код, но также и взаимосвязанные сообщества и разработчики со всего мира, работающие совместно. Используя открытую модель разработки с помощью GitHub, команды реализуют инновации и сотрудничают, выявляют уязвимости безопасности на ранних этапах и находят высококлассных специалистов. Корпоративные команды разработки могут интегрировать репозитории GitHub с Microsoft Teams и Azure. Teams — это единственное решение, объединяющее чат, голосовое взаимодействие и совместную работу с приложениями Microsoft Office, собраниями и рабочими процессами бизнеса в одном пользовательском интерфейсе. Это центр для командной работы и инноваций.

**Перейдите в [репозиторий Microsoft Azure на сайте GitHub](https://github.com/azure)** .

**Перейдите к [открытому коду в Azure](https://azure.microsoft.com/overview/choose-azure-opensource)** .

# <a name="innovation-characteristicstabcharacteristics"></a>[Характеристики инноваций](#tab/Characteristics)

## <a name="innovate-using-azure-devops"></a>Внедрение инноваций с помощью Azure DevOps

Предприятия используют GitHub для размещения исходного кода, а Azure Pipelines — для непрерывной интеграции и непрерывной поставки, позволяющих быстро развертывать изменения в Azure. Подключайте предложение Azure Boards к репозиториям GitHub, чтобы воспользоваться его широкими возможностями управления проектами, в частности канбан-досками, журналами невыполненных работ, панелями мониторинга команды и пользовательскими отчетами.

## <a name="characteristics-of-innovative-applications"></a>Характеристики инновационных приложений

С помощью Azure разработчики могут создавать ориентированные на облако приложения, используя канонические шаблоны современных приложений. Azure поддерживает и предоставляет все стандартные блоки, необходимые командам разработчиков для создания приложений, использующих контейнеры, управляемые базы данных, искусственный интеллект и бессерверные функции.

## <a name="scale-innovation"></a>Масштабирование инноваций

Предоставление инноваций для пользовательских и бизнес-приложений является приоритетным для каждого руководителя. Независимо от того, создаете ли вы мобильные функции, веб-функции, функции для Интернета вещей или смешанные реальности, инновации являются ключевыми для будущего успеха вашей организации. Чтобы масштабировать инновации, организациям необходимо сосредоточиться на "упаковке и отправке" своих уникальных бизнес-ценностей в виде составных служб, чтобы партнеры и сотрудники могли легко и эффективно участвовать в создании ценностей. У Azure есть уникальное предложение, помогающее стимулировать и масштабировать инновации в организации. В настоящее время профессиональные разработчики могут создавать приложения и стандартные блоки, которые позволяют другим экспертам в организации разрабатывать приложения на этой надежной платформе. С Azure к вашим услугам комплексная защита, предоставляемая командой экспертов, а также упреждающие средства обеспечения соответствия, которым доверяют предприятия, государственные организации и стартапы.

# <a name="liveops-innovationtabliveops"></a>[Инновации LiveOps](#tab/LiveOps)

## <a name="future-ready-liveops"></a>Подготовка к задачам будущего: LiveOps

LiveOps — это процесс разработки для непрерывного вовлечения пользователей, который ориентирован на комплексный цикл взаимодействия с клиентом. Этот подход к проектированию основывается на предоставлении ценностей для клиентов с ориентацией на удовлетворение их потребностей. LiveOps также помогает делать данные более понятными. Подход LiveOps делает данные более полезными, благодаря чему ваша организация может выполнять на основе их действия в режиме реального времени, которые повлияют на ваши приложения, клиентов и ваш бизнес в целом. Это процесс ориентирован на эксперименты, итерацию и развитие, что приводит к согласованным и частым изменениям. LiveOps — это подход к объединению DevOps с CRM. Инновационные организации применяют методы активных операций (LiveOps).

## <a name="liveops-transition"></a>Переход к LiveOps

Разработка игр — это пример отрасли, в которой LiveOps используется уже сегодня. Современные игры — это эталонные приложения с применением LiveOps. Игры переживают тихую революцию, превращаясь из готовых товаров со временем существования, измеряемым в часах, до активных сообществ со временем существования, измеряемым в годах. При наличии более чем два млрд игроков и десятков тысяч новых игр, которые поставляются каждый год, разработчики игр выбирают LiveOps, чтобы лучше привлекать и поддерживать свое сообщество с течением времени. Инновации осуществляются за счет непрерывного привлечения игрока с помощью обновлений содержимого, специальных мероприятий, рекламных акций и предложений.

## <a name="azure-playfab"></a>Azure PlayFab

[Azure PlayFab](https://azure.microsoft.com/services/playfab) — это полнофункциональная серверная платформа для игр в реальном времени и пример ведущей платформы приложений в LiveOps. Она предлагает возможность непрерывного мониторинга кода, получения аналитик и выпуска обновлений в режиме реального времени для оптимизации приложений и повышения монетизации. Цель PlayFab — предоставить интеллектуальное облако для игр, позволяющее разработчикам лучше привлекать, удерживать и монетизировать своих игроков. PlayFab предлагает дифференцированные средства и службы LiveOps более высокого уровня для повышения рентабельности и расширения срока актуальности игр. PlayFab является полностью независимой от устройства и платформ и может легко интегрироваться с поставщиками удостоверений социальных сетей, такими как Facebook или Steam, платформами хранения для управления коммерческими устройствами и платформами устройств, такими как iOS, Android, Xbox или PlayStation. Хранилище данных Azure позволяет PlayFab предоставлять интеллектуальные службы, чтобы клиенты могли совершенствовать игры и принимать лучшие решения, привлекать и удерживать игроков, а также быстрее выполнять эксперименты и итерации.

**Перейдите к [документации по Azure PlayFab](https://docs.microsoft.com/gaming/playfab/personas/liveops/liveops)** .
