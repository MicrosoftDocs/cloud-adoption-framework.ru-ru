# <a name="get-started-deliver-operational-excellence-during-digital-transformation"></a>Начало работы. доставьтесь в эксплуатацию во время цифрового преобразования

Как обеспечить эффективное использование во время цифрового преобразования? Эффективное эксплуатация — это бизнес-функция, которая напрямую влияет на решения ИТ. В оперативном состоянии основное внимание уделяется затратам на ценность клиентов и заинтересованных лиц с постоянным глазом доходов, рисков и затрат. Этот подход к управлению изменениями в Организации требует определенной стратегии, очистки результатов бизнеса и планирования управления изменениями. С точки зрения облака влияние риска и затрат осуществляется с помощью управления изменениями после внедрения и непрерывного уточнения рабочих процессов. Это может быть Автоматизация систем, практические рекомендации по управлению ИТ и согласованность ресурсов на протяжении всего жизненного цикла внедрения в облако. Действия, описанные в этом руководстве, могут помочь группе стратегии в управлении изменениями Организации, необходимой для согласованного обеспечения эффективности работы.

Работа над предприятием и портфелем начинается с одноранговых процессов стратегии и планирования, чтобы выстроить и сообщать о ожиданиях управления изменениями в Организации. Следующие шаги помогут техническим командам предоставить дисциплины, необходимые для достижения максимальной эффективности работы.

![Приступая к работе](../_images/get-started/operational-excellence-map.png)

## <a name="step-1-define-a-strategy-to-guide-digital-transformation-and-operational-excellence-expectations"></a>Шаг 1. Определение стратегии для настройки цифровых преобразований и ожидания в эксплуатации

Четкая бизнес-стратегия — это основа для любого цифрового преобразования и любых эксплуатационных усилий. Это позволяет сократить расходы и упростить ИТ-процессы, но без четкой стратегии понять, как эти изменения могут повлиять на результаты бизнеса, нацеленные на более широкие усилия по преобразованию.

**Результатам**

- Записывайте мотивации, результаты и деловое обоснование в [шаблоне стратегии и плана](https://archcenter.blob.core.windows.net/cdn/fusion/readiness/Microsoft-Cloud-Adoption-Framework-Strategy-and-Plan-Template.docx).
- Убедитесь, что метрики обучения хорошо понятны и включены в раздел "результаты бизнеса", так как эти метрики будут использоваться для обеспечения высокого качества операций и создания отчетов в ней.

**Рекомендации по поддержке готовности к завершению.**

- Основные [сведения о причинах](../strategy/motivations.md): критические бизнес-события и некоторые причины миграции обычно учитывают стоимость, что повышает важность управления затратами для всех последующих усилий. Другие посторонние решения, связанные с инновациями или ростом благодаря миграции, могут быть сосредоточены на высшей прибыли. Понимание причин помогает понять, насколько высокий уровень управления затратами на приоритет.
- [Результаты бизнеса](../strategy/business-outcomes/index.md). Некоторые финансовые результаты, как правило, чувствительны к чрезвычайно затратам. Когда желаемые результаты сопоставляются с финансовыми метриками, следует заранее привкладываться в систему управления затратами.
- [Коммерческое обоснование](../strategy/cloud-migration-business-case.md). Деловое обоснование служит обобщенным представлением общего финансового плана для внедрения в облако. Это может быть хорошим источником для первоначальных усилий по бюджетированию.
- [Обучающие метрики](../strategy/learning-metrics.md). чтобы обеспечить выравнивание между бизнес-стратегиями и более тактическихными планами управления изменениями, важно установить обучающие показатели, чтобы продемонстрировать итеративный и инкрементный прогресс в плане.

## <a name="step-2-develop-an-organizational-change-management-plan-to-span-cloud-adoption"></a>Шаг 2. Разработка плана управления изменениями Организации для охвата внедрения облака

Управление изменениями в Организации — это итеративный подход к более быстрому выровняйте людей, процессов и технологий для поддержки комплексной бизнес-стратегии. В случае высокой эффективности работы с цифровым преобразованием часто это будет по центру вокруг ИТ-плана внедрения в облаке.

**Результатам**

- Обновите [шаблон стратегии и плана](https://archcenter.blob.core.windows.net/cdn/fusion/readiness/Microsoft-Cloud-Adoption-Framework-Strategy-and-Plan-Template.docx) , чтобы отразить изменения, необходимые для реализации требуемой стратегии. Записанные изменения могут включать следующее:

  - Оценка существующего цифрового пространства.
  - План внедрения в облако, отражающий необходимые изменения и связанные с ним работы.
  - Организационные изменения, необходимые для предоставления плана.
  - План для решения навыков, необходимых для того, чтобы существующая группа успешно выполнила необходимую работу.

**Рекомендации по поддержке готовности к завершению.**

- [Сбор данных инвентаризации](../digital-estate/inventory.md): Создайте источник для анализа цифрового пространства перед внедрением.
- [Рекомендации](../plan/contoso-migration-assessment.md)по использованию службы "миграция Azure" для сбора данных инвентаризации.
- [Инкрементное обоснование](../digital-estate/rationalize.md#incremental-rationalization). во время инкрементного обоснования количественный анализ будет выявляет облачные кандидаты для целей бюджетирования.
- [Выровняйте модели затрат и модели прогнозирования](../digital-estate/calculate.md). Используйте службу "Управление затратами Azure" для согласования моделей затрат и прогнозов путем [создания бюджетов](https://docs.microsoft.com/azure/cost-management-billing/costs/tutorial-acm-create-budgets?toc=/azure/cloud-adoption-framework/toc.json&bc=/azure/cloud-adoption-framework/_bread/toc.json).
- [Создание плана внедрения в облако](../plan/plan-intro.md#build-your-cloud-adoption-plan). Создайте план с рабочей нагрузкой, активами и сведениями о временной шкале.

## <a name="step-3-manage-change-across-cloud-adoption-efforts"></a>Шаг 3. Управление изменениями в рамках деятельности по внедрению в облако

Реализация результатов бизнеса является результатом непрерывной поставки волновых раскладок внедрения. Эти волны могут включать циклы миграции и инноваций. В любом случае доставка в эксплуатацию начинается с регулярных циклов управления изменениями. Каждая волна (или выпуск в условиях Agile) предоставит набор рабочих нагрузок в облако. По мере завершения каждой волны внедрения группа облачных стратегий сможет сообщать о ходе продвижения к обучающим показателям, результатам бизнеса и общей стратегии. Аналогично, по мере завершения каждой волны внедрения, группам внедрения потребуются обновления невыполненной работы, отражающие приоритеты рабочих нагрузок в плане, в зависимости от изменений бизнес-планов и потребностей клиентов.

**Результатам**

- Непрерывное тестирование и усовершенствования стратегии и изменения плана управления на основе изменяющихся рыночных условий и завершения последних технических изменений.

**Рекомендации по поддержке готовности к завершению.**

- [Планирование выпуска](../digital-estate/approach.md): подходы к управлению изменениями с помощью планирования выпусков.
- [Поэтапное](../digital-estate/rationalize.md#incremental-rationalization)согласование: итеративный подход к управлению изменениями, нацеленный на Управление невыполненной работой по выпуску для поддержки управления волнами изменений.
- [Степень 10 подходов](../digital-estate/rationalize.md#release-planning): ограничение плана управления изменениями для детального анализа и планирование непрерывной базы с 10 рабочими нагрузками для балансировки добавочных изменений и выполнения итеративного внедрения.
- [Выровняйте пути итерации](../plan/iteration-paths.md): обновление и Добавление сведений в каждом выпуске для обеспечения текущих путей итерации.
- [Оценка рабочих нагрузок](../migrate/azure-migration-guide/assess.md?tabs=challenge-assumptions). усилия группы по внедрению в облако для оценки и работы с последним набором приоритетов миграции.
- [Согласие на ценность бизнеса](../innovate/business-value.md). усилия группы по внедрению в облако для обеспечения выравнивания ценности бизнеса в каждом выпуске новых инноваций.

## <a name="next-steps-to-delivering-operational-excellence-across-the-portfolio"></a>Дальнейшие действия по обеспечению эффективности работы в портфеле

В эксплуатацию требуется подход, оптимизированный для обеспечения надежности, производительности, безопасности и оптимизации затрат. Используйте оставшиеся инструкции по началу работы в этой серии статей, чтобы убедиться, что каждый из этих принципов реализован с помощью единообразных подходов к автоматизации.

- **Оптимизация затрат:** Непрерывная оптимизация эксплуатационных расходов с помощью руководства по началу работы по [управлению затратами на предприятия](./manage-costs.md)
- **Безопасность:** Сократите риски, интегрируя корпоративную безопасность в портфеле, используя руководство по началу работы по [обеспечению безопасности в портфеле](./security.md).
- Убедитесь, что производительность ИТ-активов поддерживает бизнес процессы, используя руководство по началу работы по [управлению производительностью](./performance.md)в масштабах предприятия.
- Повышение надежности и сокращение количества нарушений бизнеса с помощью руководства по началу работы по [реализации элементов управления для повышения надежности](./reliability.md).
