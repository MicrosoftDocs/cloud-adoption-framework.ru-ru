# <a name="get-started-build-a-cloud-governance-team"></a>Приступая к работе. Создание группы по управлению облаком

Группа управления облаком обеспечивает правильную оценку рисков и отказоустойчивость и управление ими. Эта команда обеспечивает правильную идентификацию рисков, которые не могут допускаться бизнесом. Сотрудники этой группы преобразуют риски в Управление корпоративными политиками.

![Приступая к созданию группы по управлению облаком](../../_images/get-started/governance-team-map.png)

## <a name="step-1-determine-if-a-cloud-governance-team-is-needed"></a>Шаг 1. Определение необходимости в группе по управлению облаком

Официальное руководство по облачной инфраструктуре внедрения — всегда создавать группу управления облаком. Сначала группа может быть незначительной. Однако, независимо от его размера, эта роль будет важна. Если команда не нужна, группа или пользователь в группе должны согласиться с соблюдением обязанностей, связанных с функциями управления в [облаке](../../organize/cloud-governance.md).

**Результатам**

- Определите, нужна ли вам группа по управлению облаком.
- Задокументируйте решение и ответственных лиц в [шаблоне RACI](https://archcenter.blob.core.windows.net/cdn/fusion/management/raci-template.xlsx) на вкладке Выравнивание по организации.

**Рекомендации по поддержке готовности к завершению.**

- [Функции управления облаком](../../organize/cloud-governance.md) могут быть распределены между несколькими пользователями или группами. Наличие команды, которая проходит по названию "группа по управлению облаком", имеет небольшую важность. Тем не менее, требуемые возможности должны располагаться у проверяющей стороны или группы.
- Если долгосрочная стратегия внедрения в облако компании может быть доставлена из одной целевой зоны в одной облачной среде, то объем операций по управлению и эксплуатации может быть достаточно мал для доставки одним человеком или одной командой. Эта группа вряд ли будет называться Cloud управление, так как в этом случае многие функции обслуживаются за пределами облака. Даже для этой команды приведенное ниже руководство по началу работы поможет обеспечить подготовку группы к реализации этой важной функции управления.

## <a name="step-2-align-with-other-teams"></a>Шаг 2. согласование с другими командами

Группа управления обеспечивает согласованность и соответствие набору общих политик. Эти политики берутся из текущего выравнивания с другими командами. Перед установкой политик или автоматизированного управления облаком группа управления облаком должна соответствовать другим командам, определенным в шаблоне RACI, чтобы обеспечить выравнивание по важным темам, таким как безопасность, стоимость, производительность, эксплуатация и развертывание. Шаги 4 и 5 могут помочь упростить выравнивание.

**Результатам**

- Обсуждение реализации текущего состояния и текущих планов внедрения в каждой команде.

**Рекомендации по поддержке готовности к завершению.**

- Изучите [стратегию и шаблон плана](https://archcenter.blob.core.windows.net/cdn/fusion/readiness/Microsoft-Cloud-Adoption-Framework-Strategy-and-Plan-Template.docx) вашей компании с членами группы "облачная стратегия", чтобы понять мотивацию, метрики и стратегию.
- Изучите [шаблон плана внедрения в облаке](../../plan/template.md) компании с членами группы внедрения в облако, чтобы понять временные шкалы и приоритеты.
- Изучите [книгу Operations Management](https://raw.githubusercontent.com/microsoft/cloudadoptionframework/master/manage/opsmanagementworkbook.xlsx) Team, чтобы узнать о требованиях к эксплуатации и обязательствах, которые были установлены в бизнесе.

## <a name="step-3-establish-a-cadence-with-other-teams"></a>Шаг 3. Создание ритмичности с помощью других команд

Общие сведения о внедрении облака см. в волнах (или выпусках). Регулярная ритмичность, согласованная с этими выпусками, позволит группе управления облаком проследить за рисками, которые будут представлены в следующей волновой области. Использование групп стратегии, внедрения и эксплуатации во время планирования и анализа поможет группе управления оставаться в курсе рисков.

**Результатам**

- Устанавливайте ритмичность с каждой из вспомогательных групп. По возможности выровняйте эту ритмичю циклов выпуска и планирования.
- Устанавливайте отдельную ритмичность непосредственно с помощью группы облачных стратегий (или различных членов группы), чтобы проанализировать риски, связанные со следующей волной внедрения, и оценить их уровень отклонения для этих рисков.

**Рекомендации по поддержке готовности к завершению.**

- Ознакомьтесь с [функциями управления облаком](../../organize/cloud-governance.md#deliverable) , чтобы получить дополнительные рекомендации о периодичности проведения собрания.

## <a name="step-4-review-the-methodology"></a>Шаг 4. Проверка методологии

Ознакомьтесь с методологией платформы внедрения в облаке, чтобы помочь вам в создании будущей концепции для управления и работы с этим подходом.

**Результатам**

- Получите представление о методологии, подходе и реализации, поддерживающей управляющую методологию.

**Рекомендации по поддержке готовности к завершению.**

- Изучите [управляющую методологию](../../govern/methodology.md).

**Подгруппа для учетных записей:**

- Группа по управлению облаком позволяет определить концепцию и подход к управлению.

## <a name="step-5-complete-the-governance-benchmark"></a>Шаг 5. завершение тестирования производительности системы управления

Управление — это обширный раздел. Эта короткая оценка может помочь понять, где приступить к работе.

**Результатам**

- Выполните оценку производительности управления на основе общения с различными заинтересованными лицами (или попросите других команд выполнить оценку самостоятельно).

**Рекомендации по поддержке готовности к завершению.**

- Оценка потребностей и приоритетов в управлении с помощью [теста производительности](../../govern/benchmark.md)

**Подгруппа для учетных записей:**

- Группа по управлению облаком должна понимать пропуски, выявленные в контрольном тесте управления, и обеспечивать направление на управление, помогающее устранить пропуски.

## <a name="step-6-implement-the-initial-governance-best-practice-and-configuration"></a>Шаг 6. Реализация рекомендаций и конфигурации начального руководства

Управляемая методология включает два подхода к первоначальному руководству по управлению. Проверьте каждый из них и реализуйте тот, который наиболее полно соответствует вашим потребностям.

**Результатам**

- Развертывание базовых средств управления и конфигураций Организации, необходимых для настройки среды во время следующих нескольких волновых усилий по внедрению.

**Рекомендации по поддержке готовности к завершению.**

- Ознакомьтесь с [первоначальными](../../govern/initial-foundation.md) рекомендациями по настройке и реализации рекомендаций.

**Подгруппа для учетных записей:**

- Группа управления облаком — это учетная запись для проверки и реализации рекомендаций по управлению и начального руководства.

## <a name="step-7-continuously-improve-the-governance-maturity"></a>Шаг 7. Постоянное повышение зрелости системы управления

Управление требует роста по мере завершения дополнительного облачного внедрения. Оставайтесь в курсе с текущим планом внедрения, чтобы обеспечить соблюдение принципа управления и контроля за соответствующими уровнями.

**Результатам**

- Реализуйте улучшения управления, чтобы защититься от изменяющихся рисков и потребностей в управлении.

**Рекомендации по поддержке готовности к завершению.**

- Реализуйте [Расширенные сценарии](../../govern/foundation-improvements.md) управления, чтобы улучшить первоначальное управление.

**Подгруппа для учетных записей:**

- Группа управления облаком учетная запись для согласования с текущими планами внедрения.

## <a name="step-8-evaluate-landing-zone-changes"></a>Шаг 8. Оценка изменений в целевой зоне

После развертывания и расширения зон размещения могут возникнуть новые риски или нарушения управления. Периодически просматривайте конфигурации основной зоны, чтобы выявление отклонения от политики, которые не были перехвачены средствами управления, собственными в облаке. Убедитесь, что каждое развертывание целевой зоны соответствует рекомендациям по управлению зонами.

**Результатам**

- Помогите группе облачной платформы разрабатывать усовершенствования в целевой зоне, которая соответствует политикам управления.

**Рекомендации по поддержке готовности к завершению.**

- Улучшите [Управление зоной](../../ready/considerations/landing-zone-governance.md)размещения.

**Подгруппа для учетных записей:**

- Группа управления облаком должна убедиться, что каждое развертывание размещенной зоны соответствует рекомендациям по управлению.

## <a name="step-9-adoption-handoffs"></a>Шаг 9. Внедрение хандоффс

По мере завершения новых усилий группа по внедрению в облако отправит эксплуатационные обязанности группе облачных операций и группам управления в облаке. Оставайтесь в курсе с выпусками внедрения, чтобы обеспечить правильную работу документации и выравнивания политики для рабочих нагрузок.

**Результатам**

- Регулярно просматривайте и примите хандоффс от специалистов по внедрению в облако.

**Рекомендации по поддержке готовности к завершению.**

- Создайте процесс для [адаптации новых рабочих нагрузок и ресурсов](https://docs.microsoft.com/azure/azure-resource-manager/custom-providers/concepts-resource-onboarding).

## <a name="whats-next"></a>Что дальше?

Каждая компания является уникальной и требует их управления. Выберите уровень зрелости, соответствующий вашей организации, и используйте платформу внедрения облачных решений, чтобы помочь вам получить рекомендации, процессы и инструментарий.

По мере освоения облачного управления группы могут расширить возможности работы с облаком с более быстрыми темпами. Продолжение работы по внедрению в облаке обычно приводит к активации зрелости в ИТ. Либо Разрабатывайте [группу облачных операций](./cloud-operations.md), либо синхронизируйте с группой облачных операций, чтобы обеспечить управление в рамках разработки операций.
